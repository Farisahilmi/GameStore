Berdasarkan audit mendalam, saya menemukan beberapa "hidden bugs" terkait integritas data dan privasi yang perlu diperbaiki:

### **1. Integritas Data & Database (Schema Improvements)**
*   **Duplicate Reviews**: Saat ini, seorang user bisa menulis lebih dari satu review untuk game yang sama jika memanggil API secara langsung. Saya akan menambahkan **Unique Constraint** (`userId` + `gameId`) pada model `Review`.
*   **Cascade Deletes**: Banyak relasi (seperti Review, Wishlist, Notification) tidak terhapus otomatis saat User atau Game dihapus. Saya akan menerapkan `onDelete: Cascade` pada skema database agar data sampah tidak menumpuk.
*   **Review Votes Orphan**: Saat review dihapus, data "vote" (jempol) tetap tersisa di DB. Ini akan diperbaiki dengan sistem cascade.

### **2. Perbaikan Logika Backend**
*   **Friend Activity Clarity**: Saat menerima pertemanan, pesan aktivitas "became friends with someone" terlalu ambigu. Saya akan memperbaikinya agar mencantumkan nama orang yang mengajak berteman.
*   **Activity Privacy**: Fungsi `getActivities` saat ini menampilkan aktivitas *seluruh* user di platform ke publik. Saya akan membatasinya agar lebih privat (hanya aktivitas teman atau aktivitas global yang bersifat publik).
*   **Simplified Cleanup**: Menyederhanakan kode penghapusan akun di `userController.js` dengan memanfaatkan fitur cascade dari database.
