Berdasarkan temuan saya, berikut adalah rencana perbaikan bug dan peningkatan sistem GameStore Anda:

### **1. Keamanan & Hak Akses (Security)**
*   **Voucher Security**: Membatasi akses pembuatan (`POST /`) dan daftar voucher (`GET /`) hanya untuk role `ADMIN`. Saat ini, semua user yang login bisa mengaksesnya.
*   **Validasi Input**: Menambahkan validasi skema (Joi) untuk pembuatan voucher agar data yang masuk ke database selalu valid.

### **2. Perbaikan Logika Backend (Bug Fixes)**
*   **Voucher Case-Insensitivity**: Mengubah sistem voucher agar tidak sensitif terhadap huruf besar/kecil (case-insensitive). User bisa memasukkan `welcome20` atau `WELCOME20` dengan hasil yang sama.
*   **Race Condition Check**: Memindahkan pengecekan validitas voucher ke dalam database transaction di `transactionService.js` untuk mencegah penggunaan voucher yang melebihi batas jika ada banyak transaksi bersamaan.
*   **Gifting Validation**: Menambahkan pengecekan agar pengguna tidak bisa mengirim hadiah ke diri mereka sendiri (mencegah redundansi data).

### **3. Peningkatan Frontend (UI/UX)**
*   **Voucher Input**: Mengotomatiskan konversi teks menjadi huruf kapital pada input voucher di halaman Cart.
*   **Wallet Top-up**: Menambahkan validasi minimal top-up di frontend untuk mencegah error dari server yang tidak perlu.
*   **Checkout Error Handling**: Memperbaiki tampilan pesan error saat checkout agar pengguna tahu persis kenapa transaksi mereka gagal (misalnya: voucher sudah pernah dipakai).
