# Rencana Migrasi & Publikasi (Verified)

Anda benar, sebelum dipublikasikan, kita harus memastikan kode dan database siap untuk lingkungan produksi (PostgreSQL) tanpa merusak fitur yang sudah berjalan di lokal (SQLite).

Berikut adalah rencana kerjanya:

## 1. Optimasi Schema Database (Persiapan Migrasi)
Database PostgreSQL lebih ketat daripada SQLite. Field teks panjang (seperti deskripsi game, konten forum) di PostgreSQL memiliki batas karakter jika tidak disetting khusus.
*   **Tindakan**: Saya akan memodifikasi `schema.prisma` untuk menambahkan tipe data `@db.Text` pada kolom-kolom panjang (Description, Content, Requirements, dll).
*   **Testing**: Setelah diubah, saya akan menjalankan ulang aplikasi di mode lokal (SQLite) untuk memastikan perubahan ini **tidak menyebabkan error** pada aplikasi yang sekarang.

## 2. Pembuatan Panduan Deploy (`DEPLOYMENT_GUIDE.md`)
Setelah kode terverifikasi aman, saya akan membuat panduan lengkap yang berisi:
*   **Langkah 1**: Setup Database PostgreSQL Gratis (Neon/Railway).
*   **Langkah 2**: Cara mengganti 1 baris kode (`provider = "sqlite"` ke `postgresql`) saat deploy.
*   **Langkah 3**: Cara upload ke GitHub & connect ke Vercel (Frontend) + Render (Backend).
